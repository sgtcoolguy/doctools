About This Release

Titanium SDK 5.4.0.RC is a minor release of the SDK, addressing high-priority issues from previous releases.

New Features

  • CLI-875 - appc use command should allow --prerelease option
    • The appc use command allows you to use prelease versions by using the --prerelease flag

Android Platform

Facebook

Deferred deep linking is a term used to describe one aspect of mobile deep linking. Mobile deep linking is the process of deep linking but when the end user does not have the app installed on their mobile device, they must install the app in order to access the in-app content (MOD-2193).

With this release, the SDK support's Facebook's App Invites (MOD-2200). Facebook App Invites feature allows anyone to integrate App Invites into their app when you want to invite friends to your app. When the person taps the Invite button, this feature can trigger the flow to send invites.

This release also adds the ability to set Facebook login behavior in the JavaScript API (MOD-2242).

Finally, we added support for "logPurchase" in Ti.Facebook module (MOD-2244).

Improvements patch releases

  • API-1194 - Generated Arrow Android SDK should use HttpURLConnection
    • Arrow Android SDK does not use Apache HTTP Client to make requests anymore
  • API-1221 - Expose Swagger Document for a Model or API Group
    • Provide access to Swagger doc for an individual API within Arrow apps
  • CLI-741 - Allow for refreshing of session without logging in again explicitly
    • APPC CLI allows for refreshing of a session without having to log in again
  • MOD-2248 - Ti.Facebook: Support "logCustomEvent" with optional arguments
    • Extended logCustomEvent with additional arguments in Ti.Facebook
  • TIMOB-19045 - Windows: Add -t windows to ti info command
    • Executing ti info -t windows will output information specific to Windows platform
  • TIMOB-19772 - Prompt user to accept Xcode license agreement in response to "Error 69" in debug log
    • Instead of seeing the error message "Error 69" when Xcode asks you to accept the license, you now see a prompt asking you to accept the license agreement
  • TIMOB-19802 - Android: CardView should use standard property names
    • Standardized CardView property names (e.g. removed "card" prefix)
  • TIMOB-19842: Android: CardView cardPreventCornerOverlap and cardUseCompatPadding should not start with card
    • Deprecating cardPreventCornerOverlap and cardUseCompatPadding in favor or preventCornerOverlap and useCompatPadding. Developers will see a depreciation notice.
  • TIMOB-23530 - Hyperloop - IPHONEOS_DEPLOYMENT_TARGET of 7.0 despite value of min-ios-ver in tiapp.xml
