Introduction

Starting with Release 3.2.0, the Titanium SDK no longer uses the minimum supported Android SDK (2.3.x/API level 10/Gingerbread) as the default SDK to build Android applications (if one is not specified in the tiapp.xml file). Since Release 3.2.0, Titanium will use the newest version installed if one is not specified in the tiapp.xml file. Because of this new behavior, your Android applications may look and behave differently.

This guide covers the general differences between major UI concepts on the Android platform between 2.3.x and newer versions of the Android SDK.

Specifying an SDK version

If you want to keep the appearance or behavior of a specific Android SDK, add the uses-sdk element to the Android manifest section of the tiapp.xml file with the android:targetSdkVersion attribute specifying the API level to use. For example, to use Android SDK 2.3.x (or API level 10), add the following to your tiapp.xml file:

tiapp.xml
<ti:app xmlns:ti="http://ti.appcelerator.org">
...
<android xmlns:android="http://schemas.android.com/apk/res/android">
<manifest>
<uses-sdk android:targetSdkVersion="10"/>
</manifest>
</android>
...
</ti:app>

Themes

The default theme of the Android UI changed between version 2.3.x and 3.0.x.This affects the overall appearance of the UI, changing the appearance of controls and other UI elements.For example, buttons are larger and have a flatter appearance in newer versions of the Android SDK.Android 3.0.x also introduced the built-in Holographic theme (pictured below).

With themes, you can change the overall appearance of the UI or specific elements of it, and you can define version-specific themes, allowing you to customize the appearance based on the version of Android the device is running.

For more information, seeAndroid Themes .

Android 2.3.x/Gingerbread/API level 10

Android 3.0.x/Honeycomb/API level 11 and later

For Android 2.3.x and prior, you can select to use the built-in Light (pictured below) or Black theme.

For Android 3.0.x and later, you can select to use the built-in Holo.Light (pictured below) or Holo theme.

Tabs

Android 2.3.x/Gingerbread/API level 10

Android 3.0.x/Honeycomb/API level 11 and later

In Android 2.3.x, all tabs are presented with a fixed width and crammed together below the title bar.

Since the tab group is its own view, the tab group can be animated.

In versions after Gingerbread, tabs are presented two different ways in the action bar depending on the orientation of the device. Because the tabs are part of the action bar, they are not as customizable as they were with Android 2.3.x. For example, the tab group cannot be animated since the action bar cannot animated because it is not a view; it is a window feature which is always at the top of the activity.

In portrait mode, tabs are presented as a scrollable view in the top bar area of the action bar (right below the main action bar) with the width of the tabs based on the longest title. Slide the tab bar left or right to view more tabs.

In landscape mode, tabs are presented in the view control of the action bar as a drop-down list.

Action Bar

Android 3.0.x introduces the Action Bar, adding a location for navigational and action controls.Prior to Android 3.0.x, you had to add the navigation controls to the window or use the options menu.

Android 2.3.x/Gingerbread/API level 10

Android 3.0.x/Honeycomb/API level 11 and later

Prior to the introduction of the Action Bar, Android 2.3.x devices used the options menu to display "action items". Users reveal the options menu by using a dedicated hardware menu button.

For more information, see Titanium.Android.Menu.

Starting with Android 3.0, Google introduced the Action bar, which displays the first listed action items (or menu items) in the Action Bar. Overflow items, that is, items that cannot fit in the action bar, are displayed in the overflow menu, similar to the options menu. This menu is accessed with either the overflow icon in the action bar or the dedicated hardware menu button if the device has one.

For more information, see Android Action Bar.
